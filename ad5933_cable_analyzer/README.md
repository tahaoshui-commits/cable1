# RDK X5 + AD5933 电缆通断/断路定位重构示例

这份代码不是直接沿用你原工程的“固定增益 + 一套参数全包”的写法，而是拆成两个模式：

1. 低阻模式：判断是否低阻导通
2. 电容模式：仅在非低阻时，判断是否为开路并用电容估算距离

## 文件说明

- `ad5933_driver.py`：只做底层 I2C 和原始复数采样
- `calibration_manager.py`：保存/加载校准参数
- `cable_analyzer.py`：低阻判断 + 开路定位
- `main.py`：交互菜单
- `utils_filter.py`：鲁棒统计
- `measurement_models.py`：结果数据类

## 建议的上板顺序

### 1. 安装依赖

```bash
pip install smbus2
```

### 2. 先看原始数据

```bash
python3 main.py
# 选 1
```

分别在以下三种接法下记录原始 `re / im / mag / phase_raw`：
- 低阻通路（例如 1k 标准电阻）
- 开路电缆
- 夹具悬空/仅保留测试线

### 3. 做低阻模式校准

菜单选 `2`，接标准电阻，例如 1000Ω。

### 4. 做电容模式校准

菜单选 `3`，接标准电阻，例如 100kΩ 或你实测更稳定的已知阻抗。

> 这里不建议用短路做增益校准。

### 5. 做杂散电容校准

菜单选 `4`，保持夹具接法与实测一致，但不接电缆。

### 6. 设置单位长度电容

菜单选 `5`，输入 pF/m。

如果你现在还没有精确值，可以先按已知样线做两点标定：
- 已知长度 `L1 -> C1`
- 已知长度 `L2 -> C2`

则：

```text
cable_pf_per_m = (C2 - C1) / (L2 - L1)
```

## 重要说明

1. 这版代码优先保证“架构正确”，不是保证你当前硬件参数已经最优。
2. 低阻阈值、相位阈值、CAP 模式频率需要根据你的线缆和前端再调。
3. 如果低阻短接仍显示接近 1kΩ，问题大概率不在“综合检测流程”，而在前端量程/校准方式本身。
